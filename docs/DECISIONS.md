# Design & Technical Decisions Log

Decisions made during development, with rationale. The owner's work packet
(`Lizard_Crossing_Claude_Work_Packet/`) is the source of truth where they overlap.

| # | Decision | Rationale |
|---|---|---|
| D1 | **Unity 6000.4.10f1, Built-in Render Pipeline for Phase 1** | The version installed on this machine. Built-in RP needs zero pipeline-asset configuration and suits the current style; URP migration is scheduled for the art pass (ROADMAP Phase 3). |
| D2 | **Portrait orientation, one-handed play** | Matches top mobile arcade comparables. Forward-running corridor gameplay reads best in portrait. Camera FOV is aspect-aware so cross-traffic stays visible on phones. |
| D3 | **World scale: 1 unit = 1 lizard length; world authored oversized** | Shrinking the camera to real lizard size causes near-plane, shadow, and physics precision problems. Oversizing the world gives the same fantasy with none of the issues. |
| D4 | **Free 2-axis continuous movement** (user-confirmed) | Dodging crossing footsteps demands precise lateral/backward movement and the ability to stop and wait at a lane edge for a gap. Forward bias comes from level design, not control restriction. |
| D5 | **3 hearts, hit knockback + 1.6 s invulnerability** (user-confirmed, packet HUD spec) | Kid-friendly, lets lanes be aggressive, and the hearts HUD reads instantly. Death panel after the third hit; instant retry via scene reload. |
| D6 | **Telegraphs are explicit quads (dark blob + pulsing red ring), not realtime shadow maps** | Gameplay-critical info must be 100% readable on low-end devices and art-directable (blob = true kill footprint, ring pulse = urgency). Sun shadows remain cosmetic. |
| D7 | **Procedural geometry + TextureLibrary art seam** (user-confirmed) | Phase 1 runs with zero imported assets. User-generated Higgsfield textures drop into `Assets/Resources/GeneratedArt/` (prompts in `Art/HIGGSFIELD_PROMPTS.md`) and override procedural fallbacks automatically. Real 3D models swap in during the art pass. |
| D8 | **Bugs are the soft currency** | One pickup type = clean HUD and economy; "lizard eats bugs" is fantasy-true. |
| D9 | **Stars: finish / ≥75% bugs / 100% bugs + par time** | Standard 3-star grammar; the third star demands route mastery, driving replays. |
| D10 | **Legacy input manager + custom touch layer in Phase 1** | Zero-dependency and reliable in batch setup. Wrapped behind `InputProvider` so swapping to the Input System package later touches one file. |
| D11 | **Legacy uGUI `Text` (not TextMeshPro) in Phase 1** | TMP requires an interactive import step that breaks unattended setup. All UI flows through `UIFactory`, so the TMP swap later is mechanical. |
| D12 | **Scene reload as the retry mechanism** | The whole level is procedurally rebuilt by `Bootstrap` in well under a second; reload guarantees pristine state with no reset-bug surface. |
| D13 | **Shoes render with a pant leg exiting the top of frame; no full humans** | Keeps humans mythic/huge (packet camera rule) and avoids character rigs entirely. |
| D14 | **Par time = level length / (0.78 × base speed)** | Roughly "finish without long stops"; a clean-but-careful run earns it, a greedy detour run doesn't. |
| D15 | **SFX synthesized at runtime in Phase 1** | Zero-asset rule; sub-bass thud + noise burst genuinely sells stomps. Authored audio replaces it in the polish phase via the same `Sfx` enum. |
| D16 | **Single `Bootstrap` composition root; scene contains only that object** | One place constructs and wires every system in dependency order; no scene-serialized references to drift. The Boot scene is regenerable by menu/batch. |
| D17 | **No ads/IAP code in Phase 1** (packet development rule) | The earlier `IAdService` revive stub was removed with the move to hearts. Monetization flows return in Phase 4 behind a service interface. |
| D18 | **Packet test scripts adapted to play mode** | The packet's smoke tests inspect the live scene, but this project builds its world at runtime — so the same assertions run as `[UnityTest]`s that load Boot first (`Assets/Tests/PlayMode/Phase1SmokeTests.cs`). The packet's editor validator is included verbatim and documented as "run during Play Mode". Original files remain untouched in the packet folder. |
| D19 | **Hazard cadence is per-lane (`LaneSpec`), staggered start delays** | Lanes never pulse in lockstep, difficulty ramps along the level by tightening `StepDuration`/`RespawnDelay`, and a future level = a new data table, no new code. |
| D20 | **Guaranteed minimum telegraph lead (`MinWarningLead` = 0.7 s) via a pre-launch wind-up** | Originally a footfall's warning equalled its stride flight time, so fast lanes warned too late and a careful bot still got hit at lane z=92. Each foot now holds planted with its marker already glowing for `max(bothDownPause, MinWarningLead − StepDuration)` before it launches, so warning lead is decoupled from stride speed. Verified by `AutoPlaytest`: telegraph-reading play finishes hitless; blind play is still punished in the busier back half. |
| D21 | **Meta-progression layer is data-driven and additive** (`Assets/Scripts/Meta/`) | Phase 2 backbone built without disturbing the verified Phase 1 slice. `PlayerProfile` (JsonUtility) holds all state; `MetaProgress` is the static service for economy/unlock/cosmetic/level ops and replaces the old `SaveSystem` (v1 saves auto-migrate). `LizardSpecies` (roster + `AbilityModifiers`), `CosmeticItem` (catalog), and `DailyChallenge` (date-seeded) are pure data so new content = new table rows, not code. Two currencies: **bugs** (soft, earned) and **gems** (premium, trickle-earned + future IAP). Economy rule enforced in data: every lizard/cosmetic is bug-purchasable; "Premium" only adds an optional gem path — never pay-to-win, never a gate. Selected lizard already tints the player body and grants its heart bonus; full ability + cosmetic wiring and meta UI follow. |
| D22 | **Abilities wired through one ability button; monetization via `IAdService` stub** | The roster's `AbilityModifiers` now drive gameplay: Sprint = move-speed mult (Gila), Dash cooldown mult (Gecko shortest), Camouflage = invulnerable while standing still ≥0.4 s (Chameleon), DebrisResist = +1 starting heart (Skink). The single on-screen action button is the lizard's signature move — **jump/double-jump** for Anole (with a real vertical-velocity + gravity model; airborne clears a footfall) and **dash** for everyone else. Rewarded ads are opt-in flows behind `IAdService` (`StubAdService` now, real SDK at launch): one **revive** per run (restore 1 heart + invuln, `PlayerRevived` event), **double rewards** (`MetaProgress.ApplyDoubleReward`), and **bonus chest** (`GrantBonusChest`). Verified: default-Gecko bot metrics unchanged (0/1 hits), 8/8 tests green. |
| D23 | **Meta UI is a thin uGUI view in a separate Menu scene (the app entry point)** | A new `Menu.unity` (built by `MenuBootstrap`) is build-settings index 0; `Boot.unity` (gameplay) is index 1, so the verified gameplay scene and all its play-mode tests stay isolated and unchanged. `MenuController` builds the main menu (currencies, XP, selected lizard, PLAY→Boot) plus modal overlays for the **lizard shop** (unlock/select via bug/gem flows), **cosmetics wardrobe** (slot tabs, buy/equip), and **daily** card — every action calls `MetaProgress`, so the UI holds no state. The HUD's death panel gained **Revive (ad)** + **Home**, the win panel gained the reward summary + **Double (ad)** + **Home**. A `MenuScreenshot` harness renders the menu/shop/wardrobe for review (verified: affordability gates buttons, equip/select states read correctly). |
| D24 | **Procedural premium look: color-grade post-process + tri-light lighting** | Real 3D models are gated on a Higgsfield plan, so the look is pushed as far as Built-in RP allows for free. `Grade.shader` + `CameraGrade` add a one-blit full-screen grade on the gameplay camera (contrast 1.1, saturation 1.22, warm tint, vignette 0.42) — sits below the screen-space HUD so the HUD stays crisp. Lighting upgraded to a punchier warm key + cool fill (for shoe/lizard form) + Trilight sky/ground ambient. Verified via the screenshot harness: warmer, more saturated, vignette focuses the lizard. **Build caveat:** `Grade.shader` is loaded via `Shader.Find`; add it to GraphicsSettings *Always Included Shaders* in the build-config phase so it isn't stripped on device. |
| D26 | **Verification clone unblocks in-engine batch while the owner's editor is open** | Unity locks a project by path, so the owner's open editor blocked all batch compile/test/screenshot runs. Fix: `robocopy` the project (excluding `Library/Temp/obj/Logs`) to `C:\Users\Family\LizardCrossing_verify` and run `-batchmode -runTests` there. First import is slower (no Library) but it compiles + renders independently. **Result: the redesigned gecko, seamless cobblestone, terracotta pots, Canva garden backdrop, and towering sneakers all verified rendering in-engine; 9/9 play-mode tests pass; bot metrics unchanged** (`docs/screenshots/`). Re-sync the clone with the same robocopy before each verification pass. This is the standing workaround until a Unity Editor MCP is connected. |
| D25 | **Offline asset pipeline: Canva for scenes, Python for textures + an ellipsoid previewer for geometry** | With Unity batch unavailable while the owner's editor is open, all art was produced and verified *outside* Unity. **Canva** (MCP) → logo, menu background, in-game garden backdrop (illustrated scenes — its strength; it cannot make tileable textures or 3D). **Python (Pillow/numpy)** → seamless cobblestone (`pavement_stone.png`, toroidal-Voronoi) and terracotta, verified with 2×2 tile previews; these auto-apply via `TextureLibrary` with zero code change. **A ~80-line Python ray-traced ellipsoid previewer** (`render.py`, camera matched to the low game POV) was used to redesign the procedural lizard from a "sphere-cluster" into a readable gecko (sleek body, dorsal stripe, big amber eyes, snout, toed feet, tapering tail) and verify it before porting to `LizardBody`. Port uses empty head/leg **pivots** so per-part non-uniform scales don't distort children; gait/tail/head-bob hooks preserved. Sources/licenses in `Art/ASSET_SOURCES.md`. **Unverified in-engine** (compile + render) until the editor is freed or a Unity MCP is connected. |
| D25 | **Free-asset replacement via glTFast + `ModelLibrary` (fallback-safe)** | Per owner directive (don't block on Higgsfield), a sourcing workflow picked the best free CC0/CC-BY assets per category (adversarially verified). glTFast `com.unity.cloud.gltfast@6.4.0` imports GLBs; a model drop at `Resources/Models/<key>` is auto-used by `ModelLibrary.TryBuild` (normalizes to target footprint, reseats base, strips colliders) with the procedural builder as fallback. **Landed:** Poly Pizza "Spotted Gecko" (CC-BY 3.0) as the player and "Sneakers" (CC-BY 3.0) as the hazard — both render in-game; the giant sneakers now tower over the lizard from the low POV (the core fantasy). Attribution in `ATTRIBUTION.md`. Lizard uses a code "model mode" (bob/sway/blink/squash) since the model is static; **cosmetics-on-model and a mesh-combine mobile optimization are tracked follow-ups** (imported models add many renderers). Wheels/environment/props pending (see `docs/ASSET_PIPELINE.md`). |
