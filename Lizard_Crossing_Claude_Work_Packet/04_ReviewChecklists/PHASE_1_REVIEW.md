# PHASE 1 REVIEW CHECKLIST

Use this after every Phase 1 build.

## Camera
- [ ] Camera is low enough to feel like lizard POV.
- [ ] Lizard is visible near bottom center.
- [ ] Safe zone is visible ahead.
- [ ] Hazards feel huge.
- [ ] Camera does not feel like a normal third-person camera.

## Crossing design
- [ ] Lizard moves forward.
- [ ] Main hazards move sideways across the lizard's path.
- [ ] At least one lane has left-to-right hazard motion.
- [ ] At least one lane has right-to-left hazard motion.
- [ ] The game reads like a 3D lizard POV crossing game.

## Gameplay
- [ ] Player can move/dodge.
- [ ] Player can collect bugs.
- [ ] Player can die from hazards.
- [ ] Player can reach safe zone and win.
- [ ] Game can restart or retry.

## Readability
- [ ] Warning markers/shadows appear before danger.
- [ ] Hazards are fair.
- [ ] Player understands where to go.
- [ ] HUD is readable.

## Polish
- [ ] Camera shake on impact.
- [ ] Basic particles or feedback exist.
- [ ] Environment has enough detail.
- [ ] Screenshot looks exciting.

## Technical
- [ ] No compile errors.
- [ ] No console errors during normal play.
- [ ] No duplicate scripts/classes.
- [ ] No missing prefab references.
- [ ] Code is modular.
